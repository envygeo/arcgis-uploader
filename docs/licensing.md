# Licensing approach

This repository uses the MIT License for source code and associated
documentation. See [`../LICENSE`](../LICENSE).

## Why MIT for code

The MIT License is a short, OSI-approved software licence with the SPDX
identifier `MIT`. It lets people use, copy, modify, publish, distribute,
sublicense, and sell the software, provided the copyright and permission
notices remain with copies or substantial portions of the software.

For this reference implementation, that matches the goal: other Government of
Yukon teams, vendors, partner governments, and public users can reuse the code
or port the ArcGIS workflow without negotiating a separate licence.

## How this relates to Open Government Licence - Yukon

Open Government Licence - Yukon version 2.0 is appropriate for Government of
Yukon information and data that is explicitly published under that licence. It
allows worldwide, royalty-free, perpetual, non-exclusive use of information,
including commercial use, and requires source acknowledgement. If no specific
attribution statement is supplied, the required statement is:

> Contains information licensed under the Open Government Licence - Yukon.

The Open Government Licence - Yukon is not the primary licence for this
repository's source code because it is framed around "Information" and
"Records" rather than software package reuse, and it does not grant rights to
personal information, unauthorized third-party rights, official marks, or other
intellectual property rights such as patents and trademarks. MIT is also easier
for package managers and code hosting sites to detect automatically through its
standard SPDX identifier.

## Practical rule for this office's public projects

- Use **MIT** for software source code, scripts, tests, examples, and technical
  documentation whose purpose is to explain or support the code.
- Use **Open Government Licence - Yukon** only for Government of Yukon
  information or datasets that are explicitly published as open government
  information.
- Keep official marks, logos, credentials, personal information, and third-party
  material out of repositories unless separate permission and handling rules are
  documented.
- When both code and open government information are present, keep MIT in the
  repository root `LICENSE` file and add a separate notice near the data or in
  this document identifying the Open Government Licence - Yukon material and its
  attribution statement.
